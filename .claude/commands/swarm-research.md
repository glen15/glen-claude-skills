# Swarm 리서치

여러 에이전트가 서로 다른 소스/관점에서 병렬 기술 리서치를 수행합니다.

## 절차

1. **리서치 주제 및 목표 정의**:
   - 조사 대상 기술/라이브러리/아키텍처
   - 의사결정에 필요한 핵심 질문

2. **3-4명의 리서치 팀 구성** (범용 에이전트 + 관점별 역할 부여):
   - **공식 문서 조사** (`subagent_type: general-purpose`): 최신 문서, API 레퍼런스, 마이그레이션 가이드
   - **커뮤니티 사례 조사** (`subagent_type: general-purpose`): 실제 사용 사례, 장단점, 알려진 이슈
   - **대안 비교 조사** (`subagent_type: general-purpose`): 경쟁 기술/라이브러리와 비교 분석
   - **(선택) 아키텍처 평가** (`subagent_type: architect`): 기술 선택이 시스템 설계에 미치는 영향 분석

3. **독립적으로 조사 후 발견 사항 공유**:
   - 각 팀원이 핵심 발견을 정리
   - 서로의 발견에 대해 질문/보완
   - 종합 보고서 작성

4. **최종 결론**: 추천 + 근거 + 트레이드오프

## 출력 형식

```
## 리서치 결과

### 핵심 발견
- (주요 발견 사항 요약)

### 비교 분석

| 기준 | 옵션 A | 옵션 B | 옵션 C |
|------|--------|--------|--------|
| ... | ... | ... | ... |

### 추천
(추천 옵션 + 근거)

### 트레이드오프
- (고려해야 할 사항)
```

## 참고
- 단일 기술 조사는 Sub-Agent로 충분
- 의사결정이 필요한 복잡한 비교 분석에 적합
- WebSearch, WebFetch 도구 적극 활용
