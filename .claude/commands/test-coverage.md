# 테스트 커버리지

테스트 커버리지 분석 및 누락된 테스트 생성:

1. 커버리지와 함께 테스트 실행: npm test --coverage 또는 pnpm test --coverage

2. 커버리지 보고서 분석 (coverage/coverage-summary.json)

3. 80% 커버리지 임계값 미달 파일 식별

4. 커버리지 미달 파일에 대해:
   - 테스트되지 않은 코드 경로 분석
   - 함수에 대한 유닛 테스트 생성
   - API에 대한 통합 테스트 생성
   - 중요 플로우에 대한 E2E 테스트 생성

5. 새 테스트 통과 확인

6. 이전/이후 커버리지 지표 표시

7. 프로젝트가 전체 80% 이상 커버리지 달성 확인

집중 영역:
- Happy path 시나리오
- 오류 처리
- 엣지 케이스 (null, undefined, empty)
- 경계 조건
