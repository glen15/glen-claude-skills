# 코드 리뷰

커밋되지 않은 변경사항에 대한 종합적인 보안 및 품질 검토:

1. 변경된 파일 확인: git diff --name-only HEAD

2. 각 변경 파일에서 확인할 사항:

**보안 이슈 (CRITICAL):**
- 하드코딩된 자격 증명, API 키, 토큰
- SQL 인젝션 취약점
- XSS 취약점
- 누락된 입력 유효성 검사
- 취약한 의존성
- 경로 탐색 위험

**코드 품질 (HIGH):**
- 50줄 초과 함수
- 800줄 초과 파일
- 4단계 초과 중첩
- 누락된 오류 처리
- console.log 문
- TODO/FIXME 주석
- 공개 API에 대한 JSDoc 누락

**모범 사례 (MEDIUM):**
- 변이 패턴 (불변 패턴 사용 권장)
- 코드/주석의 이모지 사용
- 새 코드에 대한 테스트 누락
- 접근성 이슈 (a11y)

3. 보고서 생성:
   - 심각도: CRITICAL, HIGH, MEDIUM, LOW
   - 파일 위치 및 줄 번호
   - 이슈 설명
   - 제안된 수정

4. CRITICAL 또는 HIGH 이슈 발견 시 커밋 차단

보안 취약점이 있는 코드를 절대 승인하지 말 것!
