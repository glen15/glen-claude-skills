# PPTX Design Principles

프레젠테이션 디자인을 위한 원칙과 가이드라인입니다. HTML → PPT 변환 또는 빌더 API 사용 시 참고하세요.

---

## Design Philosophy

**CRITICAL**: 코드 작성 전에 **대담한** 미학적 방향을 정하세요. PowerPoint 프레젠테이션은 순간적으로 경험됩니다 - 그 순간을 기억에 남게 만드세요.

**기억하세요: Claude는 탁월한 창의적 작업이 가능합니다.** 주저하지 마세요. 틀에서 벗어나 독특한 비전에 전념할 때 진정 무엇을 만들 수 있는지 보여주세요. 이것은 "그냥 슬라이드"가 아니라 시각적 스토리텔링의 기회입니다.

### Think First: Purpose and Differentiation

- **어떤 문제를 해결하는가?** 청중은 누구이며 무엇을 기억해야 하는가?
- **잊을 수 없는 한 가지는?** "전문적인 슬라이드"가 아니라 구체적인 것 - 색상 선택, 공간 패턴, 타이포그래피 보이스, 인상적인 다이어그램
- **미학적 톤 선택**: 영감을 위한 예시들:
  - **Brutally minimal**: 최대 여백, 슬라이드당 하나의 요소, 정밀한 타이포그래피
  - **Editorial/magazine**: 다중 컬럼 레이아웃, 극적인 타입 스케일, 포토저널리스틱 느낌
  - **Retro-futuristic**: 기하학적 도형, 대담한 색상, 우주 시대 낙관주의
  - **Luxury/refined**: 차분한 메탈릭, 세리프 우아함, 넉넉한 호흡 공간
  - **Industrial/utilitarian**: 모노스페이스 폰트, 그리드 구조, 명확한 실용성
  - **Art deco/geometric**: 강한 선, 대칭, 장식 패턴, 금색 악센트
  - **Organic/natural**: 자연스러운 톤, 흐르는 형태, 텍스처 배경
  - **Brutalist/raw**: 강렬한 대비, 노출된 구조, 의도적 거칠기
- **맥락이 중요**: 헬스케어 스타트업 피치 ≠ 분기별 재무 리뷰 ≠ 디자인 포트폴리오

## Taking Aesthetic Risks

**CRITICAL: 모든 프레젠테이션은 최소 하나의 대담하고 예상치 못한 디자인 선택을 해야 합니다.** 안전하고 예측 가능한 디자인은 잊힙니다. 경계를 밀어붙이세요.

### Risk-taking examples

모든 프레젠테이션에는 독특한 무언가가 있어야 합니다:

- **극단적 타이포그래피**: 120pt 헤드라인, 90° 회전 텍스트, 20px 자간의 올-캡스, 세리프 헤더와 산세리프 본문 믹스
- **예상치 못한 색상**: 파란색 대신 버건디와 금색, 코랄과 틸, 밝은 악센트의 검정 배경, 단일 팝 컬러 모노크롬
- **공간적 놀라움**: 비대칭 30/70 분할, 대각선 섹션 구분선, 겹치는 도형, 슬라이드 가장자리 텍스트 배치, 그리드 깨기
- **시각적 개성**: 두꺼운 단면 테두리(15px), L자 코너 악센트, 배경 기하학 패턴, 슬라이드 50% 컬러 블로킹
- **레이아웃 대담함**: 풀 블리드 컬러 배경, 사이드바 레이아웃, 매거진 스타일 다중 컬럼, 도형 위 플로팅 텍스트 박스

### What makes presentations forgettable (NEVER do this)

- 일반적인 "AI 슬롭" 미학: 기본 그라데이션, 모든 것 가운데 정렬, 독특한 선택 없음
- 계층이나 지배력 없는 소심하고 균등 분배된 색상 팔레트
- 어떤 주제에도 작동할 수 있는 판박이 디자인
- 모든 프레젠테이션에서 같은 안전한 선택으로 수렴

### Requirements

- ✅ 코드 작성 전 콘텐츠 기반 디자인 접근 방식 명시
- ✅ 웹 안전 폰트만 사용 (Arial, Helvetica, Times New Roman, Georgia, Courier New, Verdana, Tahoma, Trebuchet MS, Impact)
- ✅ 크기, 굵기, 색상으로 명확한 시각적 계층 생성
- ✅ 가독성 보장: 강한 대비, 적절한 텍스트 크기, 깔끔한 정렬
- ✅ 일관성: 슬라이드 전체에서 패턴, 간격, 시각 언어 반복

## Spacing, Sizing, and Information Density

**이 비율을 마스터하면 콘텐츠를 최대화하면서도 혼잡하지 않은 균형 잡힌 가독성 높은 레이아웃을 만들 수 있습니다.**

### The 4px Base Unit System

모든 간격 결정에 4px 배수를 사용하세요. 시각적 리듬을 만듭니다:

| 크기 | 용도 |
|------|------|
| **4px** | 타이트: 관련 텍스트 줄 사이 |
| **8px** | 컴팩트: 그룹 내 요소 사이 |
| **12px** | 표준: 구별되는 콘텐츠 블록 사이 |
| **16px** | 편안함: 섹션 패딩 |
| **20px** | 여유로움: 주요 분리, 슬라이드 마진 |

**일관되게 적용**: 카드 간 간격이 12px면 모든 카드 간 간격은 12px여야 합니다. 일관되지 않은 간격은 미완성처럼 보입니다.

### Font Size Hierarchy for Dense Slides

정보를 압축할 때 이 공격적인 스케일 사용:

```
Title:           32-40px (하나의 크기, 두드러짐)
Section headers: 13-15px (작지만 굵게)
Body text:       11-13px (이 크기에서도 읽기 쉬움)
Supporting text: 10-11px (세부 사항, 리스트)
Footnotes:       10px (최소 공간)
```

**규칙**: 각 레벨은 이전보다 ~2-3px 작아야 합니다. 더 큰 점프(예: 36px → 11px)는 명확한 시각적 분리(컨테이너, 색상)가 있을 때 작동합니다.

### Container Padding Rules

콘텐츠 밀도에 맞춰 패딩 매칭:

```
Dense information boxes:  12-14px padding
Standard content cards:   14-16px padding
Spacious hero sections:   20-24px padding
```

**내부 마진 패턴**: 컨테이너 내 텍스트용:

```css
margin: 0 0 4px 0;   /* Tight: 문단 내 줄 */
margin: 0 0 6px 0;   /* Compact: 리스트 항목 */
margin: 0 0 8px 0;   /* Standard: 문단 사이 */
margin: 0 0 10px 0;  /* Separated: 섹션 헤더에서 콘텐츠 */
```

### Gap vs Padding Decision Tree

- **Gap** (형제 간): 카드, 컬럼, 리스트 항목 간 간격에 사용
- **Padding** (컨테이너 내부): 박스 내 호흡 공간에 사용

```html
<!-- 올바름: 형제 간격에 gap -->
<div class="col" style="gap: 12px;">
  <div style="padding: 14px;">Box 1</div>
  <div style="padding: 14px;">Box 2</div>
</div>

<!-- 잘못됨: 부모의 gap 대신 자식에 margin -->
<div class="col">
  <div style="margin-bottom: 12px; padding: 14px;">Box 1</div>
  <div style="padding: 14px;">Box 2</div>
</div>
```

### Common Spacing Mistakes

**공간이 너무 많음**:
- 모든 요소 사이 24px+ 간격 (희소하게 보임, 공간 낭비)
- 모든 컨테이너에 20px+ 패딩 (콘텐츠가 빈 공간에 떠다님)

**공간이 너무 적음**:
- 관련 없는 항목 사이 4px 간격 (모든 것이 함께 흐려짐)
- 복잡한 컨테이너에 8px 패딩 (텍스트가 가장자리에 닿음)

**일관성 없음**:
- 여기는 12px, 저기는 16px, 또 다른 곳은 10px (미완성처럼 보임)
- 유사한 컨테이너에 다른 패딩 (시각 패턴 깨짐)

**해결책**: 간격 스케일(4, 8, 12, 16, 20)을 정하고 철저하게 지키세요.

## Typography and Content Design

**IMPORTANT: 폰트 제약**: 웹 안전 폰트만 사용하세요. PowerPoint는 안정적인 렌더링을 위해 보편적으로 사용 가능한 폰트가 필요합니다.

### 제약 내에서 독특한 선택

- **크기 대비가 파워 도구**: 극단적 크기 차이(72pt 헤더 vs 14pt 본문)가 계층과 임팩트를 만듭니다
- **굵기와 자간**: 굵게를 전략적으로 사용. 올-캡스 헤더에 자간 추가로 세련됨
- **세리프 vs 산세리프**: 신중하게 믹스 - 따뜻함/전통에 세리프, 현대적 명확함에 산세리프
- **기술 콘텐츠에 모노스페이스**: 코드, 데이터, 기술 콘텐츠에 Courier New로 정밀함 신호
- **일관성이 중요**: 최대 2-3개 서체를 선택하고 크기, 굵기, 자간으로 변주

### Content brevity - 이것은 프레젠테이션이지 보고서가 아닙니다

- 문단은 1문장, _최대_ 2문장
- 리스트당 3-5개 불릿 포인트로 제한
- 카드는 짧은 진술/단편 지원, 짧다면 완전한 문장 하나

### Visual hierarchy through typography

- 일반적으로 슬라이드당 2-3개 텍스트 크기 이하 (차트가 있는 복잡한 슬라이드는 4개 가능)
- 크기 외에 추가 구분이 필요하면 굵기(bold) 또는 불투명도 사용

## Slide Layout Zones

**960×540px 캔버스 기준:**

슬라이드는 버퍼 간격이 있는 구별된 영역으로 나뉩니다:

### 1. Title zone (상단 100px, y=0 ~ y=100)

- 슬라이드 타이틀(`<h1>`)만을 위해 예약
- 타이틀 **텍스트 박스**는 좌우 20px 패딩으로 전체 너비 (920px)
- **CRITICAL**: 전체 너비 텍스트 박스를 강제하려면 h1을 명시적 너비의 div로 감싸기:
  ```html
  <div style="width: 920px; margin: 0 20px;">
    <h1>Title Text</h1>
  </div>
  ```
- h1에 직접 너비 설정은 작동하지 않음 - 텍스트 박스가 콘텐츠에 맞게 축소됨
- 절대로 좁은 텍스트 박스에 타이틀을 배치하지 마세요 - 텍스트 줄바꿈 문제 발생

### 2. Buffer after title (10px, y=100 ~ y=110)

- 타이틀과 콘텐츠 분리를 위한 빈 공간
- 타이틀 바로 아래에서 콘텐츠 시작하면 안 됨

### 3. Content zone (380px, y=110 ~ y=490)

- 텍스트, 차트, 이미지, 카드를 위한 주요 콘텐츠 영역
- 호흡 공간을 위해 좌우 20-40px 마진 사용
- **CRITICAL**: 카드, 박스 및 기타 컨테이너는 **수평 정렬**되어야 함 - 같은 y 위치에서 시작
- 요소 간 시각적 균형과 일관된 간격 보장

### 4. Buffer before footnote (10px, y=490 ~ y=500)

- 콘텐츠와 각주 분리를 위한 빈 공간
- 콘텐츠가 하단 가장자리에 붙지 않도록

### 5. Footnote zone (하단 40px, y=500 ~ y=540)

- 각주, 출처, 면책 조항만을 위해 예약
- 각주는 **10pt 폰트 크기** 사용 (작고, 눈에 띄지 않게, 좌측 정렬)
- 이 영역에 다른 콘텐츠 불허
- 각주는 간결하게

### Example structure

```html
<body class="col bg-surface" style="width: 960px; height: 540px; position: relative;">
  <!-- Title zone: fit 클래스로 콘텐츠 높이에 맞춤 -->
  <div style="width: 920px; margin: 0 20px; padding-top: 20px;" class="fit">
    <h1 style="margin: 0;">Slide Title</h1>
    <p class="text-muted-foreground" style="margin-top: 4px;">Optional subtitle</p>
  </div>

  <!-- Content zone: fill-height로 남은 공간 차지 -->
  <div class="row gap-lg fill-height" style="padding: 10px 20px;">
    <div class="col" style="width: 45%;"><!-- Left column content --></div>
    <div class="col" style="width: 55%;"><!-- Right column content --></div>
  </div>

  <!-- Footnote zone: 고정 하단 배치를 위한 absolute positioning -->
  <p style="position: absolute; bottom: 8px; left: 20px; font-size: 10pt; color: #666; margin: 0;">
    Source: Data from Q4 2024 report
  </p>
</body>
```

## Color & Theme

**응집력 있고 대담한 미학에 전념하세요.** 날카로운 악센트가 있는 지배적 색상이 소심하고 균등 분배된 팔레트를 능가합니다. 모든 프레젠테이션은 독특한 색상 개성을 가져야 합니다.

### Core principles

- **평등보다 지배**: 한 색상이 명확히 지배(시각적 가중치의 60-70%)하고, 1-2개 보조 톤과 하나의 날카로운 악센트. 절대로 모든 색상에 동등한 가중치 주지 마세요.
- **분위기 창조**: 솔리드 화이트 배경을 기본으로 하지 마세요. 깊이와 분위기를 위해 컬러 블로킹, 그라데이션 배경, 틴티드 서피스, 극적 대비(밝은 텍스트의 어두운 슬라이드) 사용.
- **맥락별 선택**: 팔레트는 THIS 주제를 위해 디자인된 것처럼 느껴져야 합니다. 누군가 완전히 다른 프레젠테이션에 색상을 바꿔 넣어도 "작동"한다면, 충분히 구체적인 선택을 하지 않은 것입니다.

### NEVER use generic AI aesthetics

- 진부한 색상 체계 (화이트에 블루 그라데이션, 퍼플-투-핑크 그라데이션)
- 의도적 추론 없는 안전한 기업 블루
- 과다 사용된 틸, 그레이, 차분한 "전문적" 톤

### Create visual depth

- **그라데이션 배경**: 슬라이드 배경에 분위기를 위해 `linear-gradient()` 또는 `radial-gradient()` 사용
- **컬러 블로킹**: 슬라이드의 40-60%를 솔리드 색상으로 채워 시각적 구두점
- **레이어드 투명도**: 불투명도가 있는 겹치는 도형으로 깊이 생성
- **극적인 그림자**: 카드나 텍스트 박스에 대담한 드롭 섀도우로 차원 추가
- **틴티드 서피스**: 순수 화이트/블랙 대신 오프화이트, 크림, 차콜

### Build cohesion

3-5개 색상은 의도적으로 페어링된 느낌이어야 합니다 - 공유된 언더톤(모두 웜 또는 모두 쿨), 유사한 채도 레벨, 또는 의도적인 하이 콘트라스트. 팔레트는 관점을 가진 사람이 디자인한 것처럼 느껴져야 합니다.

## Visual Elements: Icons, Diagrams, Illustrations

**CRITICAL**: 시각 요소는 장식이 아니라 **커뮤니케이션**해야 합니다. 모든 아이콘, 다이어그램, 일러스트레이션은 목적이 있어야 합니다 - 개념 명확화, 관계 표시, 의미 있는 강조. 일반적 장식은 시각 요소가 없는 것보다 나쁩니다.

### When to Use Visuals

**아이콘이 적합한 경우**:
- **빠른 인식**: 한눈에 개념 표현 (설정 기어, 사용자 프로필, 경고 삼각형)
- **리스트 강화**: 불릿 포인트나 기능 목록에 시각적 앵커 추가 (최대 4-6개 항목)
- **내비게이션 큐**: 주요 섹션이나 액션으로 주의 유도
- **카테고리 마커**: 슬라이드에서 콘텐츠 유형 구분
- **상태 표시**: 상태 표시 (완료, 진행 중, 오류)

**다이어그램이 적합한 경우**:
- **프로세스 플로우**: 순차적 단계, 워크플로우, 파이프라인
- **관계**: 계층, 네트워크, 의존성, 비교
- **아키텍처**: 시스템 구성 요소, 데이터 플로우, 통합
- **타임라인**: 시간 순서, 프로젝트 단계, 마일스톤
- **비교**: 전/후, 옵션 A vs B, 기능 매트릭스

**일러스트레이션/이미지가 적합한 경우**:
- **감정적 임팩트**: 정보 전달이 아닌 감정 유발이 필요할 때
- **구체적 예시**: 실제 제품, 장소, 사람 표시
- **복잡한 개념**: 비유적 시각화가 도움이 되는 추상적 아이디어
- **브랜드 스토리텔링**: 톤과 개성 확립

### Icon Usage Patterns

**DO**:
- 전체적으로 **일관된 아이콘 세트** 사용 (스타일 혼합하지 않음 - 모두 아웃라인 OR 모두 채움, 모두 같은 굵기)
- 아이콘은 **작고 보조적**으로 유지 (일반적으로 24-48px, 강조용 최대 64px)
- 청중에게 **즉시 인식 가능한** 아이콘 선택
- **색상을 전략적**으로 사용 - 모노크롬 아이콘에 강조용 하나의 악센트 색상
- 텍스트 기준선에 아이콘 정렬하거나 카드 중앙에 배치
- 일관된 고품질 SVG 아이콘을 위해 react-icons 라이브러리 사용

**DON'T**:
- 의미 없이 순수 장식으로 아이콘 사용
- 아이콘 스타일 혼합 (아웃라인 + 채움, 다른 굵기, 다른 시각 언어)
- 아이콘이 텍스트와 주의를 두고 경쟁하게 함 (종속적으로 유지)
- 설명이 필요한 모호하거나 불명확한 아이콘 사용
- 너무 많은 아이콘으로 슬라이드 과밀화 (슬라이드당 최대 3-5개)
- 전문적 맥락에서 클립아트 스타일이나 만화 같은 아이콘 사용

### Diagram Design Principles

**복잡함보다 명확함**:
- 각 다이어그램은 하나의 주요 아이디어를 전달해야 함
- 불필요한 요소 제거 (모든 선, 박스, 화살표가 자리를 정당화해야 함)
- 시각적 계층 사용: 주요 요소는 더 크고/굵게, 보조 요소는 억제
- 다이어그램당 주요 요소 5-7개로 제한 (인지 부하 제한)

**흐름과 방향**:
- 순차적 프로세스는 왼쪽에서 오른쪽 (서양식 읽기 패턴)
- 계층은 위에서 아래
- 방사형/허브-스포크 관계는 중앙에서 바깥쪽
- 화살표는 아껴서 사용 - 레이아웃에서 방향이 명확해야 함

**시각적 일관성**:
- 같은 유형의 요소에 같은 도형 (모든 단계에 직사각형, 모든 마일스톤에 원)
- 요소 간 일관된 간격
- 균일한 선 굵기와 색상
- 근접성과 둘러싸기를 통한 명확한 그룹화

## Anti-Patterns: What NEVER to Do

잊혀지는 프레젠테이션을 만드는 이 패턴들을 피하세요:

### Typography failures

- 독특한 스케일, 굵기, 자간 선택 없이 기본 타입 크기 사용
- 한 슬라이드에 여러 폰트 패밀리 (3+ 서체 혼합 = 혼란)
- 본문 텍스트 가운데 정렬 (헤드라인만 가운데 정렬; 문단과 리스트는 좌측 정렬)
- 불충분한 크기 대비 (18pt 헤더와 16pt 본문은 거의 구별 안 됨)

### Color failures

- 일반 템플릿에 나타나는 기본 그라데이션 (특히 퍼플/블루)
- 계층이나 지배력 없이 5+ 색상을 슬라이드 전체에 균등 분배
- 저대비 조합 (화이트에 연회색 텍스트, 블랙에 다크 블루)
- 브랜드 아이덴티티가 지정되었을 때 브랜드 외 색상

### Layout failures

- 모든 것 가운데 정렬 (정적, 지루함, 흐름 없음)
- 슬라이드당 너무 많은 콘텐츠 (5개 초과 불릿, 2문장 초과 문단)
- 슬라이드 전체에 일관성 없는 간격과 정렬 (미완성처럼 보임)
- 모든 슬라이드에 정확히 같은 레이아웃 사용 (단조로움)

### Visual detail failures

- 무엇에나 적용될 수 있는 일반 클립아트 스타일 아이콘
- 미학적 방향에 기여하지 않는 장식 요소
- 반만 전념한 디자인 (하나의 화려한 슬라이드, 나머지는 기본)
- 이전 프레젠테이션과 정확히 같은 디자인 복사 (선택을 다양화하세요)

**기억하세요**: 전문적이라는 것이 일반적이라는 의미가 아닙니다. 청중과 콘텐츠에 기여하는 의도적인 선택에 전념하세요.

## Layout Tips

### 차트나 테이블이 있는 슬라이드

- **2컬럼 레이아웃 (권장)**: 전체 너비 헤더, 아래에 두 컬럼 - 한쪽에 텍스트/불릿, 다른 쪽에 주요 콘텐츠. 더 나은 균형을 제공하고 차트/테이블을 더 읽기 쉽게 만듦. 불균등 컬럼 너비(예: 40%/60% 분할)로 각 콘텐츠 유형에 맞게 공간 최적화.
- **풀슬라이드 레이아웃**: 주요 콘텐츠(차트/테이블)가 최대 임팩트와 가독성을 위해 전체 슬라이드 차지
- **절대로 수직 스택하지 말 것**: 단일 컬럼에서 텍스트 아래에 차트/테이블 배치하지 말 것 - 가독성과 레이아웃 문제 발생
